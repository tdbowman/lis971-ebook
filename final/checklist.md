# Methodology Chapter Checklist ✅

Use this checklist before submitting your final methodology chapter. Every item should be checked before you submit.

---

## Structure and Formatting

- [ ] Chapter follows a logical organizational sequence
- [ ] APA 7th edition formatting throughout
- [ ] 12–18 pages (excluding references and appendices)
- [ ] Level headings are consistent and hierarchically correct
- [ ] All tables and figures are numbered and titled per APA style
- [ ] Page numbers present
- [ ] Running head included (if required by your program)

## Philosophical Foundations

- [ ] Philosophical worldview explicitly stated
- [ ] Worldview is justified and connected to design choices
- [ ] Language throughout chapter is consistent with stated worldview

## Research Design

- [ ] Research design is named and described
- [ ] Design choice is justified with methodological literature (not just topical)
- [ ] Research questions are clearly stated
- [ ] Each research question is linked to specific methods
- [ ] Visual diagram included (especially for mixed methods)

## Sampling

- [ ] Target population is defined with clear boundaries
- [ ] Sampling strategy is named and justified
- [ ] Sample size is justified:
  - [ ] *Quantitative*: Power analysis with parameters reported
  - [ ] *Qualitative*: Saturation rationale with citations
  - [ ] *Mixed methods*: Separate justifications for each strand
- [ ] Recruitment procedures are specified
- [ ] Inclusion/exclusion criteria are stated
- [ ] Anticipated challenges and mitigation strategies are described

## Data Collection

- [ ] All data collection procedures are described in detail
- [ ] Complete instruments are included (in appendices if lengthy)
- [ ] Validity/reliability evidence is reported for instruments
- [ ] Pilot testing plan is described (if applicable)
- [ ] Data management plan is included:
  - [ ] File naming and storage
  - [ ] Backup procedures
  - [ ] Security measures
  - [ ] Retention and disposal plan

## Data Analysis

- [ ] Analytic procedures are specified step-by-step
- [ ] Each procedure is connected to a specific research question
- [ ] Software/tools are identified
- [ ] *Quantitative*:
  - [ ] Statistical tests matched to RQs and variable types
  - [ ] Assumption-checking procedures described
  - [ ] Effect sizes will be reported
  - [ ] Missing data approach specified
- [ ] *Qualitative*:
  - [ ] Coding approach named and cited
  - [ ] Coding process described step by step
  - [ ] QDA software identified
- [ ] *Mixed methods*:
  - [ ] Integration strategy specified (design, methods, interpretation levels)
  - [ ] Joint display template or plan included
  - [ ] Discordance protocol described

## Quality Criteria

- [ ] Validity/trustworthiness strategies are appropriate to your tradition
- [ ] Each strategy is described with specifics for *your* study (not generic)
- [ ] At least 3–4 quality strategies are described

## Ethics

- [ ] IRB review status identified (exempt, expedited, full)
- [ ] Informed consent procedures described
- [ ] Confidentiality protections specified
- [ ] Risk mitigation strategies described
- [ ] Data security measures detailed
- [ ] Participant compensation addressed (if applicable)

## Limitations

- [ ] Known limitations are acknowledged honestly
- [ ] Strategies for addressing or mitigating limitations are described
- [ ] Limitations are framed as boundaries, not apologies

## Citations and References

- [ ] All methodological sources are cited (not just topical literature)
- [ ] At least 2–3 sources support each major methodological decision
- [ ] Reference list is complete and formatted per APA 7th edition
- [ ] All in-text citations appear in the reference list (and vice versa)

## Final Review

- [ ] Feedback from all three checkpoint weekends has been incorporated
- [ ] Chapter has been proofread for grammar, spelling, and clarity
- [ ] A colleague or triad member has read the final draft
- [ ] Chapter reads as a coherent argument, not a disjointed checklist

---

```{admonition} One Last Thing
:class: tip
Read your methodology chapter out loud before submitting. You will catch awkward phrasing, logical gaps, and unclear passages that your eyes skip over in silent reading.
```
